# 模块职责说明

## 后端模块

| 模块 | 职责 |
| --- | --- |
| `auth` | 用户注册、登录、退出、JWT、找回密码。 |
| `user` | 用户资料、偏好、状态、免费/试用/会员状态。 |
| `membership` | 会员套餐、用户会员、后台手动调整会员、会员倒计时。 |
| `payment` | 微信支付、支付宝、订单、回调记录；银行卡/国际卡第二阶段接入。 |
| `classroom` | 班级、老师与成员关系、成员添加审核、班级学习统计。 |
| `vocab` | 词汇表、词汇条目、收藏夹、背词进度。 |
| `exercise` | 题组、句子题、排序词块、作答记录、答案校验。 |
| `dialogue` | 台词材料、台词行、词汇解析、跟读录音。 |
| `dialogue/asr` | 本地 ASR 任务领取、识别适配器、识别结果对比和回写。 |
| `matching` | 词汇连连看游戏、成绩记录。 |
| `stats` | 学习行为、每日统计、学习汇总、排行榜。 |
| `admin` | 后台账号、角色、权限、操作日志、后台配置；老师后台人员也使用后台账号登录，通过 `teacher_admin` 角色获得班级和报表相关权限。 |

## 前端后台模块

| 模块 | 职责 |
| --- | --- |
| `dashboard` | 用户、会员、订单、活跃度和内容数量概览。 |
| `users` | 用户查询、状态查看、会员调整入口。 |
| `memberships` | 会员套餐、会员记录、订单状态、异常订单快捷视图、订单处理记录、异常订单处理、支付回调审计。 |
| `classes` | 班级、老师、成员、审核、班级统计。 |
| `content` | 词汇、题库、台词材料、多语言内容。 |
| `media` | 音频、图片、用户录音等媒体资源。 |
| `reports` | 学习统计、正确率、排行榜和班级报表。 |
| `system` | 后台账号、角色权限、支付、ASR、上传限制等系统配置。 |

## 学生端模块

| 模块 | 职责 |
| --- | --- |
| `home` | 功能入口、试用/会员状态。 |
| `membership` | 套餐、支付、订单状态、倒计时。 |
| `practice` | 听录音排序、听写、拼音写句子、译文排序。 |
| `vocab` | 背单词、收藏、进度。 |
| `dialogue` | 台词听写、排序、跟读和识别结果。 |
| `matching` | 连连看设置、游戏、成绩。 |
| `classroom` | 班级列表、邀请码加入、成员信息；学生端不提供创建班级、审核成员或查看班级学习统计。 |
| `stats` | 学习记录、连续天数、正确率、排行榜。 |

## 手机端模块

| 模块 | 职责 |
| --- | --- |
| `shell` | UniApp H5 优先，底部导航包含首页、背单词、练习、班级和记录；登录页不放入主导航。 |
| `auth` | 复用学生端 JWT 协议，401 时清理 Token 和用户资料并返回登录页。 |
| `home` | 移动端学习概览、试用/会员状态和常用功能入口。 |
| `vocab` | 词汇表列表、词卡学习、读音音频播放、收藏和进度保存；免费用户可直接进入。 |
| `practice` | 句子练习题集和作答；移动端直达时需先校验试用/会员权限并展示锁定状态。 |
| `dialogue` | 台词材料列表、台词播放、排序/听写/跟读练习、录音入口和词汇解析。 |
| `classroom` | 班级列表、邀请码加入和成员信息；移动端学生视角不提供老师侧管理能力。 |
| `stats` | 学习汇总、每日统计和学习事件分页。 |

## 支付适配器约定

- `payment` 模块的订单、会员权益和事务逻辑放在 `PaymentService`，线上支付和后台线下支付登记都复用同一套会员开通逻辑。
- 支付渠道参数生成、回调解析和验签放在 `PaymentProviderAdapter` 实现中。
- `offline` 为后台登记线下支付渠道，不走第三方回调，也不作为赠送会员使用；赠送或补偿仍应走用户会员调整流程。
- `dev` 环境默认启用 `PAYMENT_MOCK_ENABLED=true`，用于本地验证创建订单、支付成功和会员开通闭环。
- 未接入官方 SDK 前，可用签名载荷适配器验证回调：回调 JSON 中携带 `orderNo`、`amount`、`status`、`providerTradeNo` 和 `signature`；签名为去掉 `signature` 后按 key 升序拼接 `key=value` 并使用 `notify-secret` 做 HMAC-SHA256。
- 正式微信支付、支付宝 SDK 接入时，只替换或新增适配器实现，不把渠道细节写进 Controller 或业务 Service。
