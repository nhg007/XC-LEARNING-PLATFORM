# 背词进度设计

## 目标

背单词进度分为两层：

- `user_vocab_progress` 保存用户在某个词表里的当前位置，服务于继续学习、进度条和首页概览。
- `user_vocab_item_progress` 保存用户对某个词条的学习状态，作为“该单词是否已学”的准确信息来源。

## 单词级状态

`user_vocab_item_progress.status` 使用以下状态：

- `learning`：正在学习，尚未标记为已学。
- `learned`：已学过该词。
- `reviewing`：已学过，进入复习状态。
- `mastered`：已掌握该词。

没有进度记录的词条视为未学，不单独落库。

## 背词页行为

用户在背词页点击“已学，下一词”时：

1. 前端提交当前词条 ID、下一词位置和学习耗时。
2. 后端校验词条属于当前词表且处于启用状态。
3. 后端写入或更新 `user_vocab_item_progress`，默认将该词状态设为 `learned`。
4. 后端按单词级进度重新统计已学数量，并同步更新 `user_vocab_progress.reviewed_count`。
5. 后端记录一条 `study_events` 背词事件，用本次学习耗时计入每日统计。

词表排序变化、插入新词或停用词条时，单词级已学状态不依赖位置推断，避免把旧进度对应到错误词条。
