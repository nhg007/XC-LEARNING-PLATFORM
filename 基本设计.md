# 汉语学习网站 基本设计

## 1. 文档概要

### 1.1 目的

本文档基于 `需求分析.pdf` 整理，用于描述汉语学习网站的基本设计，包括系统概要、功能模块划分、主要页面、数据表设计、权限规则和接口初稿。后续可在此基础上继续补充详细设计、原型图、接口字段定义和测试用例。

### 1.2 需求范围

系统面向学习汉语的用户，提供中、俄、英三种语言界面，并支持用户注册、7 天免费试用、会员付费解锁、学习练习、背单词、台词训练和词汇连连看等功能。补充范围包括管理员后台手动调整会员状态和会员有效期、会员时长按月或自定义天数配置，以及班级功能与班级内学习情况查看。银行卡/国际卡支付不纳入首期实现，作为后续扩展项处理。

### 1.3 设计假设

- 网站采用 Web 应用形式，支持桌面端和移动端浏览器。
- 账号注册后默认获得 7 天全功能免费试用。
- 试用结束且未付费时，仅保留“背单词”功能。
- 付费会员在会员有效期内可使用全部功能。
- 管理员可在后台手动开通、延长、缩短或取消用户会员，并设置会员开始时间和结束时间；前台按剩余有效期自动倒计时。
- 系统内容以汉语为学习目标，俄语和英语主要用于界面语言、翻译和释义。
- “听录音排列成句”和“听录音写句子”的录音内容确定为汉语句子本身。
- 支付渠道首期确定为微信支付和支付宝；银行卡/国际卡支付不纳入首期，可在第二阶段按目标国家/地区再接入。
- 语音识别确定为本地部署服务，不依赖第三方云端识别。
- 需要学习统计报表，包括连续学习天数、正确率和排行榜。
- 不需要教师或管理员手动给学生分配词汇表或题组，用户自行选择学习内容；班级功能只用于成员组织和学习情况查看。
- 支持班级功能，老师可直接添加成员；其他班级成员发起添加成员申请时，需要老师审核通过后生效。老师可查看本班成员的学习时间、做题数量、正确率和最近学习时间等统计。
- 管理后台为首期必须实现范围，用于维护题库、词汇、媒体、台词材料、会员、班级和报表。

## 2. 系统概要

### 2.1 系统定位

本系统是一个面向汉语学习者的在线学习平台。用户可以选择界面语言，并在不同练习中选择英语或俄语作为翻译或释义语言。系统通过题目练习、词汇卡片、台词跟读和小游戏帮助用户进行听力、拼写、词汇和语音训练。

### 2.2 用户角色

| 角色 | 说明 |
| --- | --- |
| 游客 | 未登录用户，只能访问登录、注册、语言选择、公开说明页面。 |
| 试用用户 | 注册后 7 天内，可使用全部学习功能。 |
| 免费用户 | 试用过期且未付费，只能使用背单词功能和账号基础功能。 |
| 会员用户 | 已付费且会员有效，可使用全部学习功能。 |
| 老师 | 创建和维护班级，直接添加成员，审核其他成员提交的加人申请，查看本班成员学习统计。 |
| 管理员 | 维护用户、词汇表、题库、音视频材料、翻译和会员配置。 |

### 2.3 核心能力

- 用户账号：注册、登录、退出、找回密码、账号资料。
- 会员权限：7 天试用、试用过期限制、付费会员解锁、后台手动调整会员状态和有效期、会员剩余时间倒计时。
- 多语言：网站界面支持汉语、俄语、英语，并记住用户选择。
- 偏好保存：记录界面语言、翻译语言、释义语言和音效开关。
- 练习题型：
  - 听录音排列成句。
  - 听录音写句子。
  - 看拼音写句子。
  - 看俄语或英语译文排列汉语句子。
  - 听电视剧、短视频或动画片录音写台词、排列台词、跟读台词。
- 背单词：支持 HSK1-6、YCT1-6、分类词汇、专业词汇和收藏夹。
- 连连看：基于词汇表或收藏夹生成 4x4、7x7、10x10 难度游戏。
- 学习记录：记录练习结果、背词进度、收藏词汇、游戏成绩、连续学习天数、正确率和排行榜。
- 班级管理：创建班级、老师直接添加成员、普通成员提交加人申请、老师审核申请、查看班级成员学习时间、做题数量、正确率和最近学习时间。
- 内容管理：维护题目、词汇、音频、台词、翻译和解析。

### 2.4 会员权限规则

| 用户状态 | 判断条件 | 可用功能 |
| --- | --- | --- |
| 游客 | 未登录 | 登录、注册、公开页面 |
| 试用中 | 注册时间未超过 7 天，且无有效会员也可 | 全部功能 |
| 免费用户 | 试用已过期，且无有效会员 | 背单词、收藏夹、账号设置、购买会员、班级基础查看 |
| 会员用户 | 存在有效会员记录 | 全部功能 |

权限判断建议封装为统一服务，前端负责隐藏不可用入口，后端接口必须再次校验。

管理员手动调整会员后，系统应立即以最新 `user_memberships` 记录为准重新计算权限。前台会员页显示试用或会员剩余时间倒计时，倒计时到期后自动刷新用户权限状态。

### 2.5 技术架构建议

| 层级 | 技术选型 | 说明 |
| --- | --- | --- |
| 前台 Web | Vue3 + Element Plus + TypeScript | 面向浏览器访问，负责学习、练习、会员、班级和个人中心。 |
| 手机端 | Vue3 + UniApp | 面向移动端，可发布为 H5、小程序或 App，支持移动端学习、录音和班级查看。 |
| 管理后台 | Vue3 + Element Plus + TypeScript | 首期必须实现，负责题库、词汇、媒体、会员、订单、班级、报表和系统配置。 |
| 后端 API | Spring Boot 3 + MyBatis Plus | 负责账号、权限、会员、支付、学习业务、内容管理和统计，数据访问层使用 MyBatis Plus。 |
| 数据库 | PostgreSQL | 保存业务数据、内容数据、订单和统计数据。 |
| 缓存 | Redis | 保存登录状态、验证码、排行榜缓存和热点数据。 |
| 文件存储 | MinIO / 对象存储 | 保存音频、录音、封面、导入文件等媒体资源。 |
| 本地语音识别 | 独立 ASR 服务 | 内网部署，通过后端异步调用，识别学生跟读录音。 |
| 网关与部署 | Nginx + Docker | 统一转发前台、后台、API、文件访问和 ASR 服务。 |

### 2.6 端应用划分

| 应用 | 访问对象 | 主要职责 |
| --- | --- | --- |
| Web 前台 | 学生用户/老师 | 登录注册、会员购买、学习练习、背单词、连连看、学习记录、班级管理。 |
| 手机端 | 学生用户/班级成员 | 移动端学习、背单词、练习、录音跟读、查看记录和班级统计。 |
| 管理后台 | 管理员 | 用户、会员、订单、班级、词汇、题库、台词材料、媒体、统计报表维护。 |
| 后端服务 | 前台/手机端/后台 | 统一提供业务 API、权限校验、支付回调、会员调整、班级统计计算和 ASR 调用。 |

### 4.2 背单词页面规则

- 进入时先选择词汇表。
- 卡片正面只显示汉字。
- 点击卡片后翻转，显示当前释义语言下的翻译。
- “显示拼音”按钮点击后显示拼音。
- 喇叭按钮播放该词汉语读音。
- 星星按钮收藏或取消收藏。
- 自动记录当前词汇表背诵位置。
- 页面底部显示当前进度，例如 `11/46`。
- 不保留“跟踪进度”和右下角无关功能。

### 4.3 连连看页面规则

- 进入游戏前选择词汇来源：词汇表或收藏夹。
- 难度支持 4x4、7x7、10x10。
- 用户匹配汉字和对应英语或俄语释义。
- 匹配成功播放成功音效，匹配失败播放失败音效。
- 支持关闭音效，并记住用户选择。
- 页面左上角提供返回菜单。
- 不需要右上角暂停功能。

### 4.4 班级页面规则

- 登录用户可创建班级或通过邀请码加入班级。
- 老师可通过用户邮箱、用户 ID 或邀请码直接添加成员，添加后立即生效。
- 普通成员可提交添加成员申请，申请状态为待审核，必须由老师审核通过后才生效。
- 班级详情展示成员列表、学习时间、做题数量、正确率和最近学习时间。
- 普通成员只能查看自己的学习统计和班级基础信息。
- 老师可审核加人申请、移除成员，成员可主动退出班级。
- 班级不负责分配词汇表或题组，学习内容仍由用户自行选择。

## 5. 业务流程设计

### 5.1 注册与试用流程

1. 用户填写注册信息。
2. 系统创建用户账号。
3. 系统写入用户偏好默认值。
4. 系统根据注册时间计算试用截止时间，默认注册后 7 天。
5. 用户进入首页，可使用全部功能。
6. 每次访问受限功能时，系统检查是否试用中或会员有效。

### 5.2 试用过期流程

1. 用户登录或访问功能时，系统计算试用是否过期。
2. 若试用已过期且无有效会员，则标记为免费用户。
3. 首页学习功能仅开放背单词入口，其他受限学习功能显示会员提示；账号、购买会员和班级基础入口仍可访问。
4. 用户可进入会员页购买会员。

### 5.3 练习作答流程

1. 用户选择练习类型和翻译语言偏好。
2. 系统返回题目、汉语录音、词块、拼音、答案校验规则等数据。
3. 用户作答并提交。
4. 系统校验答案，返回正确/错误和错误位置。
5. 用户可继续修改。
6. 用户点击查看答案时，系统展示正确答案。
7. 用户点击翻译时，系统按选择语言展示俄语或英语翻译。
8. 系统保存作答记录。

### 5.4 台词跟读流程

1. 用户选择材料和训练模式。
2. 系统播放台词音频。
3. 用户录音跟读。
4. 系统保存录音并调用语音识别。
5. 系统展示识别出的汉字文本。
6. 系统对比标准台词，提示差异。
7. 系统展示正确答案、词汇解析和翻译。

### 5.5 背单词流程

1. 用户选择词汇表或收藏夹。
2. 系统读取用户在该词汇表的上次位置。
3. 页面显示当前词卡。
4. 用户查看拼音、播放读音、翻转释义或收藏。
5. 用户切换下一词时，系统保存当前进度。

### 5.6 会员支付流程

1. 用户在会员页选择套餐。
2. 用户选择微信支付或支付宝。
3. 后端创建支付订单，订单状态为 `pending`。
4. 后端调用对应支付渠道生成支付参数或二维码。
5. 用户完成支付后，支付渠道调用后端回调接口。
6. 后端校验回调签名和金额，更新订单为 `paid`。
7. 后端创建或延长用户会员有效期。
8. 前端轮询或刷新订单状态，展示会员已开通。

### 5.7 学习统计流程

1. 用户完成练习、背词、台词训练或连连看时，系统写入学习行为数据。
2. 系统按天汇总学习时长、练习次数、正确次数、背词数量和游戏成绩。
3. 系统根据每日学习记录计算连续学习天数。
4. 系统根据作答记录计算正确率。
5. 系统按统计指标生成排行榜，可按日、周、月或总榜展示。

### 5.8 后台内容维护流程

1. 管理员登录后台。
2. 管理员维护词汇表、词汇条目、题组、句子题、台词材料和媒体资源。
3. 管理员上传汉语录音、词汇读音、台词音频等资源。
4. 管理员填写英语/俄语翻译、释义和词汇解析。
5. 管理员发布内容后，前台和手机端可读取对应学习内容。

### 5.9 后台手动调整会员流程

1. 管理员在后台查询用户。
2. 管理员选择开通、延长、缩短或取消会员。
3. 系统要求填写会员开始时间、结束时间、调整原因和可选备注。
4. 系统写入或更新会员记录，并记录后台操作日志。
5. 用户下次访问或刷新会员页时，系统展示最新会员状态和剩余时间倒计时。

### 5.10 班级创建与统计流程

1. 老师或普通用户创建班级；系统记录创建者，但成员添加权限按班级角色判断。
2. 老师通过邮箱、用户 ID 或邀请码添加成员时，系统直接写入班级成员关系，成员状态为 `active`。
3. 非老师成员发起添加成员申请时，系统写入待审核申请，状态为 `pending_teacher_review`。
4. 老师审核通过后，成员状态更新为 `active`；老师拒绝后，状态更新为 `rejected`。
5. 用户完成练习、背词、台词训练或连连看时，系统继续写入学习行为数据。
6. 老师进入班级详情页，按成员查看学习时间、做题数量、正确率和最近学习时间。
7. 班级统计从用户学习行为和每日汇总中聚合，不影响用户自行选择词汇表或题组。

## 8. 权限与状态设计

### 8.1 后端权限判断

受限功能接口统一检查：

- 用户是否已登录。
- 是否在试用期内。
- 是否存在有效会员。
- 如果不是试用中或会员，只允许访问背单词相关接口、账号/支付接口和班级基础接口。
- 班级统计接口必须校验班级成员身份；普通成员只能查看自己的学习统计，老师可查看本班成员统计。
- 班级加人接口必须校验操作者班级角色；老师添加成员直接生效，非老师成员提交的加人申请必须进入待审核状态，由老师审核。

### 8.2 前端显示规则

- 试用中：显示全部功能，提示试用剩余天数。
- 会员中：显示全部功能，提示会员有效期。
- 免费用户：背单词功能可点击，其他功能显示锁定状态和升级入口。
- 游客：点击学习功能时跳转登录或注册。
- 登录用户显示班级入口；班级详情中的成员统计和成员审核入口按班级角色控制可见范围。

## 9. 非功能设计

### 9.1 多语言

- 界面文案通过 i18n 字典管理。
- 用户选择的界面语言保存到 `user_preferences.ui_language`。
- 练习翻译语言、词汇释义语言、连连看释义语言分别保存，避免每次登录重新选择。
- 听录音类题目的音频统一为汉语句子录音。

### 9.2 音频与录音

- 系统音频统一通过 `media_assets` 管理。
- 用户录音上传后也保存为媒体资源。
- 台词跟读中的语音识别结果保存到 `speech_records`。
- 浏览器端需要请求麦克风权限，并提供录音、停止、试听、重新录制功能。
- 本地 ASR 服务通过内网 API 调用，后端创建 `asr_jobs` 异步识别任务。
- ASR 服务只接收用户录音文件地址或文件流，识别完成后回写识别文本和任务状态。

### 9.3 答案校验

- 排序题按词块 ID 顺序校验。
- 输入题建议进行基础标准化后校验，例如去除首尾空格、统一中文标点。
- 错误提示应尽量显示错误位置，但不直接覆盖用户答案。
- 查看答案应记录到 `exercise_attempts.showed_answer`。

### 9.4 安全

- 密码只保存哈希，不保存明文。
- 微信支付、支付宝回调必须校验第三方签名、订单号、金额和支付状态。
- 后端必须校验会员权限，不能只依赖前端隐藏入口。
- 后台手动调整会员必须记录管理员、调整前后时间、原因和操作日志。
- 班级统计涉及成员学习数据，必须校验班级关系，避免跨班级查看其他用户数据。
- 普通成员提交的加人申请在老师审核前不得出现在正式成员统计中。
- 上传文件限制类型、大小和访问权限。
- 后台管理端需要独立管理员账号、角色和操作日志。

### 9.5 可扩展性

- 新题型可通过 `exercise_type` 扩展。
- 新词汇表可通过 `vocab_lists.list_type` 和 `level` 扩展。
- 新语言可扩展 `content_translations` 和 i18n 字典。
- 会员时长可通过套餐的 `duration_unit`、`duration_value` 和 `duration_days` 支持按月、按天或自定义配置。
- ASR、微信支付、支付宝应封装为独立适配器，便于后续替换或扩展；银行卡/国际卡支付作为第二阶段独立适配器接入。

### 9.6 统计报表

- 学习行为先写入 `study_events`，再汇总到 `user_daily_stats` 和 `user_learning_summary`。
- 连续学习天数以用户所在时区的自然日计算。
- 正确率按 `正确次数 / 作答次数` 计算，查看答案不计为正确。
- 排行榜可定时生成到 `leaderboard_entries`，避免实时大范围计算。
- 班级统计按 `class_members` 聚合成员的学习行为和每日汇总，支持查看使用时间、做题数量、正确率和最近学习时间。

### 9.7 部署与运维

- 前台 Web、后台管理端和 API 服务通过 Nginx 统一入口访问。
- 手机端 UniApp 的 H5 版本可复用同一 API，若发布小程序需适配登录和支付流程。
- ASR 服务建议独立部署，避免识别任务占用主业务服务资源。
- 媒体文件建议存放在 MinIO 或对象存储，并通过权限控制访问用户录音。

## 11. 首期开发建议

建议第一阶段实现以下范围：

1. Web 前台：Vue3 + Element Plus，实现账号、首页、会员、练习、背单词、连连看、学习记录和班级管理。
2. 手机端：Vue3 + UniApp，实现登录、背单词、基础练习、录音跟读、学习记录和班级查看。
3. 管理后台：Vue3 + Element Plus，实现后台登录、用户、会员手动调整、订单、班级、词汇、题库、媒体、台词材料和报表管理。
4. 后端 API：Spring Boot 3 + MyBatis Plus，实现注册登录、7 天试用、会员权限、会员后台调整、微信/支付宝支付、班级接口、班级成员审核、内容接口和统计接口。
5. 基础练习：实现听汉语录音排列成句、听汉语录音写句子、看拼音写句子、看英/俄译文排列汉语句子。
6. 背单词：实现词汇表选择、词卡翻转、显示拼音、播放读音、收藏夹和自动保存进度。
7. 连连看：实现词汇表/收藏夹来源、4x4/7x7/10x10 难度、音效开关和游戏记录。
8. 台词训练：实现材料选择、听录音、台词听写、跟读录音、本地 ASR 识别结果展示、答案和词汇解析。
9. 学习统计：实现连续学习天数、正确率、背词数量、游戏成绩、班级统计和排行榜。
10. 部署基础：实现 PostgreSQL、Redis、MinIO、Nginx、业务后端和本地 ASR 服务的基础部署。

第二阶段建议完善内容批量导入、运营报表、银行卡/国际卡正式接入、支付退款、ASR 评分优化、排行榜反作弊和更多学习数据分析。
